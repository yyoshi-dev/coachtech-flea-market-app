## US002: ユーザーはログインができる

<th>機能詳細（完了状態）</th>
</tr>

<tr>
<td>FN007</td>
<td>ログイン認証機能</td>
<td>使用技術 : fortify</td>
</tr>

<tr>
<td>FN008</td>
<td>入力フォーム</td>
<td>
必要な情報<br>
1. メールアドレス<br>
2. パスワード
</td>
</tr>

<tr>
<td>FN009</td>
<td>バリデーション</td>
<td>
使用技術：formrequest<br>
バリデーションルールは別シート「基本設計書」を参照
</td>
</tr>

<tr>
<td>FN010</td>
<td>エラーメッセージ表示</td>
<td>

　　1. メールアドレスを入力してください<br>
　　2. パスワードを入力してください<br>

　　1. ログイン情報が登録されていません
</td>

<td>

2. ログイン画面から会員登録画面に遷移できること
</td>
</tr>

<tr>
<td>FN012</td>
<td>メールを用いた認証機能（応用）</td>
<td>
1. 新規会員登録にメール認証を行う<br>
2. 新規会員登録後にメール認証をしないでログインを試みた場合はメール認証誘導画面へ遷移<br>
3. メール認証誘導画面内の「認証はこちらから」ボタンを押下するとメール認証画面へ遷移<br>
4. メール認証を実装した際の画面遷移<br>
　a. 初回会員登録<br>
　b. メール認証誘導画面<br>
　c. メール認証画面<br>
　d. プロフィール設定画面<br>
5. メール認証を行うのは会員登録時、初回ログイン時<br>
6. 使用技術：mailhogかmailtrap
</td>
</tr>

<tr>
<td>FN013</td>
<td>認証メール再送機能</td>
<td>メール認証誘導画面で「認証メールを再送する」ボタンをクリックすると認証メールを再送信される</td>
</tr>

</tr>

2. 送付先住所変更画面にて登録した住所が商品購入画面に反映されている<br>
